# Code of Conduct

This repository aims for direct, respectful, evidence-based collaboration around Azure DevOps Server and TFS workflows.

## Expected behavior

- Be specific about the problem, server assumptions, and observed behavior.
- Keep discussion technical, factual, and scoped to the repository.
- Sanitize logs, commands, screenshots, and examples before posting them publicly.
- Respect documented support boundaries instead of presenting unsupported behavior as if it were guaranteed.

## Unacceptable behavior

- Harassment, discrimination, intimidation, or personal attacks
- Publicly posting secrets, tokens, private hostnames, or exploit details that put others at risk
- Spam, repeated off-topic promotion, or deliberately disruptive discussion
- Knowingly misrepresenting test evidence, server behavior, or support claims

## Scope

This code of conduct applies to issues, pull requests, reviews, discussions, and other project spaces managed through this repository.

## Enforcement

Repository maintainers may edit, hide, lock, or remove content that violates these expectations and may restrict participation when needed to protect the project and its contributors.

## Reporting

For conduct concerns, contact the repository maintainer through GitHub. For active vulnerabilities, use the private reporting path described in [SECURITY.md](SECURITY.md).
